🔄 Data Flow Architecture

📱 Frontend → Backend → Cloud

1️⃣ Student Registration Flow

  • Student fills registration form
  • Data sent to Flask API endpoint /api/register
  • Backend validates and stores in MongoDB Atlas
  • Face data uploaded to AWS S3 bucket
  • Confirmation sent back to frontend

2️⃣ Attendance Marking Flow

  • Camera captures student face image
  • Image sent to /api/recognize endpoint
  • AWS Rekognition/Google Vision API processes face
  • Match found in cloud database
  • Attendance record created and synced to cloud

3️⃣ Data Synchronization

POST /api/attendance/manual { "studentId": "CS2024001", "studentName": "John Doe", "subject": "Computer Science" }

🔗 RESTful API Endpoints

👥 Student Management

  • POST /api/register - Register new student with face data
  • GET /api/students - Retrieve all registered students

✅ Attendance Operations

  • POST /api/recognize - Face recognition attendance
  • POST /api/attendance/manual - Manual attendance entry
  • GET /api/attendance - Fetch attendance records

📊 Reports & Analytics

  • POST /api/reports/generate - Generate custom reports
  • POST /api/export - Export data to cloud storage
